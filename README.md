# PRODAJA_WEB

Projekat maloprodaje

Важна напомена коју сам раније нашао (сачувано у меморији пројекта): корак 5 провјерава само да ли је Sumatra успјешно предала посао spooler-у — не и да ли је NRG физички одштампао страницу. Ако би spooler касније задржао job (нпр. нема папира, štampač offline, паузиран ред), апликација би и даље показала "успјешно" иако ништа није изашло. То је засебан проблем у PRINT_AGENT пројекту (споменуо сам ти рјешење раније — polling статуса print job-а), нисам га дирао јер си рекао да ћеш то пренијети другом девелоперу.

Дакле: тренутно, ако видиш "NRG" у header-у, job иде тачно тамо. Треба само да провјериш физички да ли NRG заиста испљуне рачун када кликнеш штампу.
